Implement three exported functions:

1. ValidateTemplate(wsAbs, filePath string) error
   - layoutPath = filepath.Join(wsAbs, "templates", "layouts", "base.gohtml")
   - Call template.ParseFiles(layoutPath, filePath) — two args, layout first, same as ssr.go
   - On error return: fmt.Errorf("%s: template parse error: %w", filepath.Base(filePath), err)
   - This function has no I/O beyond the ParseFiles call (no os.ReadFile needed here)

2. ValidateHTML(filePath string, content []byte) error
   - Package-level var templateDirectiveRE = regexp.MustCompile(`(?s)\{\{.*?\}\}`) — MUST use (?s) dotall flag for multi-line directives (per RESEARCH Pitfall 2)
   - Package-level var voidElements = map[string]bool covering all HTML5 void elements: area, base, br, col, embed, hr, img, input, link, meta, param, source, track, wbr
   - stripped = templateDirectiveRE.ReplaceAll(content, []byte(" "))
   - z = gohtml.NewTokenizer(bytes.NewReader(stripped))
   - var stack []string; var errs []string
   - Loop: z.Next() switch on token type
     - gohtml.ErrorToken: if z.Err() == io.EOF → goto done; else append tokenizer error, goto done
     - gohtml.StartTagToken: get rawName, selfClose = z.TagName(); if !selfClose && !voidElements[name] → push name to stack
     - gohtml.EndTagToken: get name; if len(stack)==0 → append unexpected close error; else if stack[top] != name → append mismatch error; else pop stack
   - done: append unclosed-tag error for each remaining stack entry
   - If len(errs)>0: return fmt.Errorf("%s: HTML structure:\n  %s", filepath.Base(filePath), strings.Join(errs, "\n  "))

3. ValidateFiles(wsAbs string, files []string) error
   - var errs []string
   - For each f: os.ReadFile(f); if err → append read error, continue
   - Call ValidateTemplate(wsAbs, f); if err → append
   - Call ValidateHTML(f, content); if err → append
   - If len(errs)>0: return fmt.Errorf("validation failed:\n%s", strings.Join(errs, "\n"))
