1. ## SSH Key Scope
   Use a dedicated SSH key for agent workspace access. Restrict key via SSH authorized_keys command= or directory restrictions to workspace subdirectory only. Never use a personal or shared key. Key compromise: revoke immediately and rotate.

2. ## PocketBase Admin Token
   The PocketBase admin API token (from /api/collections/_superusers/auth-with-password) grants full collection management. Store in environment variable or secrets vault (e.g., POCKETBASE_ADMIN_TOKEN). Never commit token strings to git. Token scope: collection management only — do not use superuser credentials for content API calls. Rotate on any suspected compromise.

3. ## Git History Safety
   workspace/.pb_data/ contains the SQLite database and must NOT be committed. Add workspace/.pb_data/ to .gitignore immediately after monms init. Never commit .env files, credential files, or token strings. If a secret is accidentally committed, rotate it immediately and use `git filter-branch` or BFG to purge history.

4. ## Operator Escape Hatches
   - View commit history: git -C workspace log --oneline
   - Revert a bad agent commit: git -C workspace revert <commit-sha>
   - Manual rollback (whole working tree): git -C workspace checkout -- .
   - Pre-commit hook bypass (emergency only): git commit --no-verify (WARNING: skips validation — use only to unstick a broken hook, never in normal agent workflow)
   - View staged changes before commit: git -C workspace diff --cached
