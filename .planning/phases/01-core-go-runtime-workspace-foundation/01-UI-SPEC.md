# Phase 1 — UI Design Contract

> Visual and interaction contract for the MonMS workspace scaffold: base layout, index stub, error pages, and hybrid CDN + `main.css` styling. Generated by gsd-ui-researcher. Phase 1 ships structure only — auth badge and inline editing activate in Phase 3.

**Sources:** D-21, D-22, D-23, D-24 (01-CONTEXT.md); DEMO-03, WRK-05, D-17–D-20; RESEARCH.md §D Tailwind CDN; PRD §5 base layout (structure reference only).

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Go SSR + Go HTML templates — not React) |
| Preset | not applicable |
| Component library | none — semantic HTML + `main.css` component classes |
| Icon library | none in Phase 1 (inline SVG hamburger in index stub only) |
| Font | system-ui stack via Tailwind preflight (`ui-sans-serif, system-ui, sans-serif`) |
| Styling approach | Hybrid: Tailwind v3 Play CDN (preflight + minimal utilities) + `workspace/assets/main.css` (branded components) |
| JS runtime | Alpine.js 3.14.8 (mobile nav demo), HTMX 1.9.12 (CDN present; no active swaps in Phase 1) |

**shadcn gate:** Skipped — greenfield Go monolith, no `components.json`, no React build pipeline per PROJECT.md constraints.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge dot padding, inline flex gaps |
| sm | 8px | Nav item padding, compact card padding |
| md | 16px | Default element spacing, button padding-x |
| lg | 24px | Card padding, section inner spacing |
| xl | 32px | Layout horizontal padding (mobile), grid gaps |
| 2xl | 48px | Section vertical padding |
| 3xl | 64px | Hero vertical padding, page-level top spacing |

**Tailwind mapping:** Use utility classes `p-4`, `p-6`, `py-12`, `px-6`, `gap-2`, `gap-4`, `space-y-6` in templates — all resolve to this scale via Tailwind defaults.

**Exceptions:**
- Mobile nav hamburger touch target: 44×44px minimum (`p-3` on 20px icon = 44px) — accessibility exception to strict 8pt grid
- Editor overlay placeholder (Phase 3): `fixed top-4 right-4` positioning uses 16px offset from viewport edge

## Typography

| Role | Size | Weight | Line Height | CSS / Class |
|------|------|--------|-------------|-------------|
| Body | 16px | 400 (regular) | 1.5 | `text-base`, `.hero__body`, `.site-header__logo`, card `<h2>` |
| Label | 14px | 600 (semibold) | 1.4 | `text-sm font-semibold`, nav links, error code label, `.btn`, `.card code` |
| Heading | 24px | 600 (semibold) | 1.2 | `text-2xl font-semibold`, `.hero__title`, error page `<h1>` |

**Four sizes only in Phase 1:** 14px, 16px, 24px — no 13px, 18px, or 36px anywhere in templates or `main.css`.

**Weights allowed in Phase 1:** 400 and 600 only. Do not introduce 500 or 700.

**Font smoothing:** Rely on Tailwind preflight defaults. No custom `@font-face` in Phase 1.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f8fafc` (slate-50) | `<body>` background, page canvas |
| Secondary (30%) | `#ffffff` (white) + `#f1f5f9` (slate-100) | Cards (`.card`), header bar, mobile nav drawer background |
| Accent (10%) | `#4f46e5` (indigo-600) | Primary buttons, text links on hover, focus rings, future live-editor badge |
| Text primary | `#0f172a` (slate-900) | Body copy, headings |
| Text muted | `#475569` (slate-600) | Secondary paragraphs, error descriptions |
| Border | `#e2e8f0` (slate-200) | Card borders, header bottom border, nav divider |
| Destructive | `#dc2626` (red-600) | Reserved — no destructive UI in Phase 1 |

**60/30/10 application:**
- **60%** — `body` uses `bg-slate-50 text-slate-900`; main content area inherits dominant surface
- **30%** — `.card`, `.site-header`, mobile nav panel use white/slate-100 surfaces against the slate-50 canvas
- **10%** — accent appears only on: `.btn--primary`, inline text links (`text-indigo-600 hover:text-indigo-700`), focus-visible outlines; **not** on body text, nav labels at rest, or error page headings

**Accent reserved for:** primary CTA buttons, "Go home" link emphasis on error pages, nav link hover/active states, editor overlay badge (Phase 3 placeholder uses `hidden` — do not render accent in Phase 1)

## Base Layout Structure

**File:** `workspace/templates/layouts/base.gohtml`

### Template contract

```gohtml
{{define "base"}}
<!DOCTYPE html>
<html lang="en">
<head>
  <!-- meta, title block, CDN + main.css — see CDN Placement -->
</head>
<body class="bg-slate-50 text-slate-900 min-h-screen">
  <!-- editor overlay placeholder — hidden in Phase 1 -->
  <!-- site header with nav slot -->
  <main class="max-w-6xl mx-auto px-6 py-12">
    {{template "body" .}}
  </main>
  <!-- HTMX auth header script placeholder — commented out until Phase 3 -->
</body>
</html>
{{end}}
```

### Document regions (top to bottom)

| Region | Element | Classes / behavior |
|--------|---------|-------------------|
| Head | `<head>` | charset, viewport, title block, CDN scripts, `/assets/main.css` |
| Editor overlay | `#editor-overlay` | `hidden` in Phase 1; Phase 3 replaces with `{{if .IsLoggedIn}}` badge |
| Site header | `<header class="site-header">` | Sticky top, white bg, bottom border; contains logo + nav |
| Main | `<main>` | `max-w-6xl mx-auto px-6 py-12` — content slot |
| Footer | none in Phase 1 | Omit |

### Page template contract

Every page template (`index.gohtml`, `errors/errors.gohtml`, etc.) defines:

```gohtml
{{define "title"}}Page Title{{end}}
{{define "body"}}...{{end}}
```

Handler executes: `tmpl.ExecuteTemplate(w, "base", data)`.

### Template context (Phase 1)

| Key | Type | Phase 1 value |
|-----|------|---------------|
| `IsLoggedIn` | bool | always `false` — do not branch on auth in Phase 1 templates except commented placeholders |
| `Code` | int | error pages only — 404 or 500 |
| `Message` | string | error pages — human-readable message |
| `Path` | string | error pages — attempted URL path |

## CDN & Script Placement

**Order in `<head>` (strict):**

1. `<meta charset="UTF-8">`
2. `<meta name="viewport" content="width=device-width, initial-scale=1.0">`
3. `<title>{{block "title" .}}MonMS{{end}}</title>`
4. `<script src="https://cdn.tailwindcss.com"></script>`
5. Inline Tailwind config:
   ```html
   <script>
     tailwind.config = {
       corePlugins: { preflight: true },
       theme: { extend: {} }
     }
   </script>
   ```
6. `<link rel="stylesheet" href="/assets/main.css">`
7. `<script src="https://unpkg.com/htmx.org@1.9.12"></script>`

**End of `<body>` (before closing tag):**

8. `<script defer src="https://cdn.jsdelivr.net/npm/alpinejs@3.14.8/dist/cdn.min.js"></script>`
9. HTMX auth script — **comment block only** in Phase 1:
   ```html
   {{/* Phase 3: inject pb_auth Bearer header for HTMX requests
   <script>...</script>
   */}}
   ```

**Rationale:** HTMX in head (PRD pattern); Alpine deferred at body end to avoid blocking paint; Tailwind before `main.css` so component classes can override preflight where needed.

## Site Header & Alpine Mobile Nav

**File:** `workspace/templates/index.gohtml` (header duplicated inline for Phase 1 stub; extract to fragment in later phases if needed)

### Desktop (≥768px)

- Horizontal nav: `hidden md:flex gap-6`
- Links: Home (`/`), Admin (`/_/`)
- Logo/site name: **MonMS** — links to `/`, `.site-header__logo` (16px semibold)

### Mobile (<768px)

- Hamburger button: `md:hidden`, 44×44px touch target, `aria-expanded`, `aria-controls="mobile-nav"`
- Alpine state: `x-data="{ open: false }"` on `<header>` or wrapper
- Toggle: `@click="open = !open"`, `:aria-expanded="open.toString()"`
- Panel: `#mobile-nav`, `x-show="open"`, `x-transition`, full-width dropdown below header
- Close on navigate: `@click="open = false"` on each mobile link
- `x-cloak` + `[x-cloak] { display: none !important; }` in `main.css` to prevent flash

### Nav link styling

- Default: `text-slate-600 hover:text-indigo-600 text-sm font-semibold`
- Active (home on index): `text-indigo-600`

**Alpine wiring proof:** Mobile open/close must work without page reload. No other Alpine components in Phase 1.

## Error Page Styling

**File:** `workspace/templates/errors/errors.gohtml`

### Layout

Centered error card within main content area:

```gohtml
{{define "title"}}{{.Code}} — {{if eq .Code 404}}Not Found{{else}}Error{{end}}{{end}}
{{define "body"}}
<section class="error-page">
  <p class="error-page__code">{{.Code}}</p>
  <h1 class="error-page__title">{{if eq .Code 404}}Page not found{{else}}Something went wrong{{end}}</h1>
  <p class="error-page__message">{{.Message}}</p>
  <a href="/" class="btn btn--primary">Go home</a>
</section>
{{end}}
```

### Visual spec

| Element | Style |
|---------|-------|
| `.error-page` | `text-center`, max-width 480px, mx-auto, py-16 |
| `.error-page__code` | 14px semibold, `text-slate-400`, letter-spacing wide |
| `.error-page__title` | 24px semibold, slate-900, mb-4 |
| `.error-page__message` | 16px regular, slate-600, mb-8, word-break break-word |
